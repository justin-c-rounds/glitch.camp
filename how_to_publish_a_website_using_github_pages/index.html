How to Publish a Website Using GitHub Pages

GitHub's Pages feature is a great way to publish a website. You can do it just using a browser, or with GitHub Desktop, or using Git on the command line.

Creating GitHub Pages in a Browser

  1. If you don't have one already, create an account on GitHub.

  2. Once you have a GitHub account, create a new repository for your project.

  3. In your project, create a new file (or upload an index.html file and skip to step 5).

  4. Name the file index.html and add some HTML into the editor.

  5. Scroll to the bottom of the page, write a commit message, and commit the new file.

  6. Click on the Settings tab and scroll down to the GitHub Pages section. Then select the master branch source and click on the Save button.

  7. Load your project website in a browser by going to http://YOUR-USERNAME.github.io/YOUR-REPOSITORY-NAME/.

  8. Add additional pages, CSS, and media files as necessary.
